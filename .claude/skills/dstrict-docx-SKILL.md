# d'strict Word 문서 제작 지침

이 스킬은 d'strict 내부 문서 표준을 정의합니다. 기존 `docx` 스킬의 기술적 구현 방법(docx-js, XML 편집 등)은 그대로 사용하되, 아래 서식 규칙을 **최우선**으로 적용합니다.

---

## 1. 파일명 규칙

모든 파일은 `YYMMDD_제목` 형식으로 생성합니다.

- 예: `260318_업무요청서_이메일_스팸방지.docx`
- 회사명이 파일명에 포함될 경우 소문자 d로 시작: `dstrict` (파일명에서는 어퍼스트로피 제외)
- 문서 본문 내 회사명 표기: `d'strict` (소문자 d, 어퍼스트로피 포함)

---

## 2. 폰트 규칙

### 기본 폰트 매핑

| 문자 종류 | 폰트 | 비고 |
|-----------|------|------|
| 한국어 (국문) | 맑은 고딕 (Malgun Gothic) | 기본 |
| 영문 / 숫자 | Arial | 기본 |

### 자간 (Character Spacing) 규칙

| 문자 종류 | 자간 설정 | 비고 |
|-----------|----------|------|
| 한국어 | 좁게 (Narrow/Condensed) -0.6pt | 기본 10pt 기준 |
| 영문 / 숫자 | 보통 (Normal) 0pt | 항상 보통 |

**폰트 크기에 따른 국문 자간 조정:**

| 폰트 크기 | 국문 자간 |
|-----------|----------|
| 10pt 이하 | 좁게 -0.6pt |
| 11~16pt | 좁게 -0.6pt ~ -1.0pt |
| 17~21pt | 좁게 -1.0pt |
| 22~27pt | 좁게 -1.5pt |
| 28~35pt | 좁게 -2.0pt |
| 36pt 이상 | 좁게 -2.5pt |

> **중요**: 한국어와 영문/숫자가 혼합된 문장에서는 반드시 별도의 TextRun 객체를 사용하여 각각 독립적으로 자간을 적용해야 합니다. 하나의 TextRun에 한국어와 영문을 섞지 마십시오.

### docx-js 구현 예시

```javascript
// 한국어 텍스트 (좁게 -0.6pt = -12 half-points)
new TextRun({
  text: "업무요청서",
  font: "Malgun Gothic",
  size: 20, // 10pt
  characterSpacing: -12, // -0.6pt in half-points (1pt = 20 half-points)
}),
// 영문/숫자 텍스트 (보통)
new TextRun({
  text: "CEO Report",
  font: "Arial",
  size: 20, // 10pt
  // characterSpacing 생략 또는 0
}),
```

## 3. CI 로고 규칙

- 문서 내 회사명을 표시할 때는 텍스트 대신 CI 로고 이미지를 사용합니다.
- 밝은 배경: `dstrict_CI_BLACK.png`
- 어두운 배경: `dstrict_CI_WHITE.png`
- 로고 파일은 uploads 폴더에서 확인합니다.

---

## 4. 수정 작업 원칙

- 기존 작업에서 별도로 수정 언급이 없었던 부분은 **컨펌된 것**으로 간주합니다.
- 추가 수정 요청 시, 요청된 부분**만** 수정하고 나머지는 변경하지 않습니다.
- 임의로 기존 컨펌 내용을 변경하지 않습니다.

---

## 5. 문서 품질 원칙

- 확실하지 않은 정보는 사실처럼 기재하지 않습니다.
- 데이터나 수치가 포함된 경우 출처와 근거를 명시합니다.
- 작업 완료 후 오류가 없는지 교차검증합니다.
- 애매한 판단이 필요한 경우 선택지를 제시하고 CEO 결정을 받습니다.

---

## 6. 체크리스트

문서 생성/수정 완료 시 아래 항목을 확인합니다:

- [ ] 파일명이 `YYMMDD_제목` 형식인가?
- [ ] 국문 폰트가 맑은 고딕인가?
- [ ] 영문/숫자 폰트가 Arial인가?
- [ ] 국문 자간이 좁게(-0.6pt 이상)로 설정되었는가?
- [ ] 영문/숫자 자간이 보통(0pt)인가?
- [ ] 한영 혼합 문장에서 별도 TextRun으로 분리했는가?
- [ ] 회사명이 d'strict (소문자 d)로 표기되었는가?
- [ ] CI 로고가 텍스트 대신 이미지로 삽입되었는가? (해당 시)
- [ ] 기존 컨펌 내용이 임의로 변경되지 않았는가? (수정 작업 시)
